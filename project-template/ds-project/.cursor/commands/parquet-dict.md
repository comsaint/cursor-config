請使用 DuckDB MCP 完成 parquet 資料字典撰寫。

我會在這個 command 後面提供 1 個或多個 parquet 路徑。
每行一個路徑；也可能是 glob，例如：
- C:/path/to/file.parquet
- C:/path/to/folder/*.parquet

任務要求：

1. 必須真的使用 DuckDB MCP，不要只憑猜測回答。

2. 對我提供的每一個 parquet / parquet group：
   - 先取得 schema
   - 再看少量樣本資料
   - 再做基礎 profiling

3. schema 檢查請優先使用：
   DESCRIBE SELECT * FROM read_parquet('<path>');
   如有需要，再用 parquet_schema('<path>') 補充 metadata。

4. 對每個資料集盡量補充：
   - row count
   - 欄位名稱
   - DuckDB 推定型別
   - nullable 判斷
   - example values
   - 日期欄位候選
   - 數值欄位候選
   - 類別欄位候選
   - ID / key 欄位候選
   - flag 欄位候選
   - 需要人工確認的欄位

5. 若輸入包含多個 parquet：
   - 先各自建立資料字典
   - 再補一節 cross-dataset notes
   - 說明欄位命名是否一致
   - 哪些欄位看起來可作 join key
   - 哪些欄位語義相近但名稱不同
   - 哪些欄位可能存在粒度不一致問題

6. 最後輸出完整的 DATA_DICTIONARY.md 草稿，格式如下：

# DATA_DICTIONARY.md

## Overview
- Input parquet(s):
- Generated by:
- Confidence notes:

## Dataset: <dataset_name>

| column_name | inferred_type | nullable | candidate_role | example_values | guessed_description | needs_human_confirmation | notes |
|---|---|---:|---|---|---|---|---|

## Cross-dataset notes
- dataset A vs dataset B:
- possible join keys:
- possible grain mismatch:
- unclear relationships needing confirmation:
