## 4) 標準流程清單（先建立骨架）

### RB-001 重跑資料管線（Pipeline Re-run）
- 目標：重建指定期間資料產物。
- 風險：重跑範圍錯誤導致覆蓋不該覆蓋資料。
- 必填：執行區間、輸入來源、輸出驗證結果。

### RB-002 更新特徵（Feature Update）
- 目標：新增/調整特徵後重建特徵表。
- 風險：特徵洩漏或 schema 破壞相容性。
- 必填：特徵版本、相容性檢查、模型影響評估。

### RB-003 回補資料（Backfill）
- 目標：補齊缺漏歷史資料。
- 風險：重複寫入、時間窗錯位、統計偏移。
- 必填：回補範圍、去重策略、對帳結果。

### RB-004 失敗排查（Failure Triage）
- 目標：定位並處理 pipeline / 訓練失敗。
- 風險：處理順序錯誤導致 MTTR 變長。
- 必填：錯誤分類、根因、暫時修復與長期修復。

## 6) 例行演練與維護

- [ ] 每月抽測至少 1 條關鍵流程（含回滾）。
- [ ] 每次重大事故後 48 小時內更新對應 runbook。
- [ ] 每季檢查 owner、指令、路徑、門檻是否過期。
- [ ] 新人 Onboarding 使用 runbook 實際演練並回饋可讀性。

## 7) 參考最佳實務（摘要）

- Runbook 要「可操作」且「可驗證」，每一步都需有預期輸出。
- 文件必須單一權威來源，避免多版本衝突。
- 定期演練與版本維護和撰寫同等重要；未演練 runbook 等於未準備。
- 內容應假設讀者不熟悉系統，降低夜間或交接時的人為風險。
